Explanation

This tracks the progress of the microsoft/STL repo. For more info, read the Changelog.

This chart has two vertical axes. For all lines, fewer is better.

  • Left axis:
    • Open bugs. Note: we're still porting bugs from our Microsoft-internal database to GitHub.
    • Skipped tests in the libcxx test suite. There are many reasons for skipped tests: STL bugs, test bugs, compiler bugs, missing STL features, missing compiler features, etc. Note: to avoid double-counting, this chart excludes skipped tests due to missing STL features.
  • Right axis:
    • C++17 STL features remaining to be implemented. This reached zero when we implemented <charconv>, C++17's final boss.
    • C++20 STL features remaining to be implemented. This increases when more features are voted into the Working Paper at WG21 meetings.
    • LWG issue resolutions remaining to be implemented. This also increases after each WG21 meeting.

We don't count "patch papers" as extra work when one paper adds a feature and another paper modifies that feature before we've implemented it.

This chart weights features equally, instead of weighting by usefulness for users or difficulty for implementers.
